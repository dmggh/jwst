The editing context is the context where new references are typically added as part of routine maintenance. Normally the editing context will track files which are not yet operationally available. The editing context always reflects the last context generated by any submission... even the other context modes like operational or recent.

The operational context is reported back from the pipeline to CRDS as the context currently being used for processing. This choice might be used to perform a point-fix for a problem affecting operations without pushing out all the changes in the editing context. Note that any context produced by using this choice immediately becomes the editing context for followon edits. The operational context is defined by the pipeline and mirrored by CRDS so it will not change despite submissions against it. For multi-stage fixes it should only be used for the first submission.
