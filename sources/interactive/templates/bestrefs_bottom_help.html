NOTE: This page is a diagnostic aid and not for production data calibration. {% if observatory|upper == "HST" %} For production use for HST, the CRDS command line tools are used to update dataset headers.

On-site at STScI typically no file downloads or configuration are required for running crds.bestrefs. Best references can be assigned to HST dataset FITS files like this: 

    % setenv iref, jref, etc. to  /grp/crds/cache/references/hst for processing.
        
    % python -m crds.bestrefs --update-bestrefs --files *.fits

For Offsite use (without ssh or VPN) you can create a personal copy of required rules and references and run crds.bestrefs like this: 

    % setenv CRDS_PATH $HOME/crds_cache
    % setenv CRDS_SERVER_URL https://hst-crds.stsci.edu
    % setenv iref, jref, etc. to  ${CRDS_PATH}/references/hst for processing.
        
    % python -m crds.bestrefs --update-bestrefs --files *.fits --sync-references=1

This sets up a personal cache containing the latest operational rules and server configuration information. Adding --sync-references=1 downloads any required references to your CRDS cache automatically. If you have another source of reference files, syncing and using CRDS references is optional.

For Off Site use without VPN or ssh, you can create a cache of required rules and references with settings like these: 

    % setenv CRDS_PATH $HOME/crds_cache                         # small scale personal cache
    % setenv CRDS_SERVER_URL https://jwst-crds.stsci.edu
    % ... run calibration steps ...
